# 🛠️ bidernet v2.8.3-php - תיקון שגיאת DB

## 🚨 התיקון

תוקנה שגיאה: `Column not found: 1054 Unknown column 'isCarousel'`

### מה היה הבעיה?
ה-API ניסה לשמור פוסטים עם השדות החדשים (`isCarousel`, `mediaItems`) ב-DB **שעדיין לא כלל את העמודות האלה**.

### מה תוקן?
ה-`api.php` עכשיו עושה 2 דברים בצורה אוטומטית:

1. **Auto-migration** - בודק אם העמודות החדשות קיימות, ומוסיף אותן אוטומטית אם לא
2. **Defensive INSERT** - גם אם המיגרציה נכשלת (למשל הרשאות), ה-INSERT לא יישבר - הוא יזהה אילו עמודות קיימות וישתמש רק בהן

**התוצאה**: שמירת פוסטים תעבוד תמיד! 🎉

## 💡 איך זה עובד?

בכל פעם שה-API נטען:
1. בודק אם העמודה `isCarousel` קיימת ב-DB
2. אם לא - מוסיף אותה (`ALTER TABLE`)
3. בודק אם העמודה `mediaItems` קיימת
4. אם לא - מוסיף אותה
5. ממשיך כרגיל

**בטוח**: מתבצע רק `ADD COLUMN IF NOT EXISTS` - לא נוגע בעמודות קיימות.
